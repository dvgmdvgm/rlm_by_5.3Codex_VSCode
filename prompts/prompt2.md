мы переходим на продвинутый уровень Agentic Workflow. Я хочу настроить систему автономной оркестрации субагентов (Planner, Worker, Code Reviewer) в моем проекте, чтобы ты мог автономно выполнять большие задачи, разбивая их на шаги. 

Важнейший нюанс: у нас уже подключен внешний MCP-сервер RLM памяти. Этот инструмент обращается к моей локальной RLM-системе, которая содержит бесконечную память всего моего проекта. 

Твоя задача — сгенерировать для меня структуру конфигурационных файлов IDE (для папки `.github/` или аналогичной системы правил). Все системные инструкции (содержимое файлов) пиши **СТРОГО НА АНГЛИЙСКОМ ЯЗЫКЕ** для экономии токенов и лучшего понимания.

Пожалуйста, выдай мне содержимое для следующих 5 файлов с точным указанием директорий:

#### 1. Субагент: Planner (Файл: `planner.md`)
Напиши системный промпт для Планировщика. 
**Его задачи:**
- Получить большую задачу от пользователя.
- ОБЯЗАТЕЛЬНО использовать инструмент RLM, чтобы запросить старый контекст, архитектурные паттерны или стили проекта, прежде чем составлять план.
- Разбить задачу на мелкие, логичные шаги.
- Создать markdown-файлы для каждой подзадачи в папке `.vscode/tasks/` и создать главный файл с планом (master plan).

#### 2. Субагент: Worker (Файл: `worker.md`)
Напиши системный промпт для Воркера.
**Его задачи:**
- Брать в работу одну конкретную подзадачу из папки `.vscode/tasks/`.
- Если для написания кода ему нужно вспомнить конкретные CSS-классы, старые функции или логику, он ДОЛЖЕН сначала вызвать RLM (отправить туда точечный запрос), получить точный контекст и только потом писать код.
- Писать чистый, рабочий код и внедрять его в проект.

#### 3. Субагент: Code Reviewer (Файл: `code_reviewer.md`)
Напиши системный промпт для Ревьювера.
**Его задачи:**
- Проверять код, только что написанный Worker'ом.
- Искать баги, уязвимости, проблемы с оптимизацией и несоответствие архитектуре.
- Если есть критичные ошибки, возвращать список проблем (REJECT) для Worker'а. Если все идеально — одобрять (APPROVE), чтобы система перешла к следующей задаче.

#### 4. Скилл: Оркестрация (Файл: `/skills/orchestrate_workflow/skill.md`)
Напиши инструкцию для процесса оркестрации. Это главный цикл (Workflow).
**Логика скилла:**
- Когда поступает большая задача, сначала запускается `planner`.
- Как только `planner` создал файлы задач, запускается цикл для каждой задачи из плана:
  1. Запускается `worker` для выполнения задачи.
  2. Запускается `code_reviewer` для проверки работы воркера.
  3. Если ревьювер нашел ошибки, снова запускается `worker` для их исправления.
  4. Как только ревьювер дает APPROVE, цикл переходит к следующей задаче.
- Процесс повторяется, пока все задачи не будут выполнены.

#### 5. Команда запуска (Файл: `/commands/orchestrate.md`)
Напиши простую команду, которая будет триггерить этот процесс.
**Логика команды:**
- Команда называется `/orchestrate`.
- В инструкции команды должно быть жестко прописано: "Всегда используй скилл `orchestrate_workflow` для выполнения запроса пользователя".
